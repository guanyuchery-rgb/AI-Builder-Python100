LLM / RAG 放在 ML、DL、RL 后面，更容易理解 embedding、检索、评估和上下文。

## 已吸收的旧 Day 材料

旧 `Day81-Day85` 的 LLM Report Assistant 和旧 `Day91-Day95` 的本地 RAG 笔记助手，不再作为独立迁移材料保留，统一吸收到 Project05：

- `Day81 LLMReportAssistant骨架` -> `IC35 文档数据集构造`、`IC39 RAG上下文打包`。
- `Day82 上下文打包与Prompt` -> `IC39 RAG上下文打包`、`IC40 引用回答`。
- `Day83 结构化解析与审查规则` -> `IC41 结构化输出`、`IC43 错误案例库`。
- `Day84 评估集与样例报告` -> `IC42 LLM评估集`。
- `Day85 交付、复盘与公开说明` -> `IC45 README与截图`、`IC46 RAG总交付`。
- `Day91 RAG笔记助手骨架` -> `IC35 文档数据集构造`。
- `Day92 索引与检索层` -> `IC36 文本清洗与切分`、`IC37 Embedding地图`、`IC38 向量检索`。
- `Day93 回答Prompt与来源引用` -> `IC39 RAG上下文打包`、`IC40 引用回答`。
- `Day94 RAG评估与错误案例` -> `IC42 LLM评估集`、`IC43 错误案例库`。
- `Day95 交付与复盘` -> `IC46 RAG总交付`。

Project05 的主线不是“做一个大系统”，而是逐步沉淀：文档数据、切分、embedding、检索、上下文、引用、结构化输出、评估、错误案例和最终交付。
