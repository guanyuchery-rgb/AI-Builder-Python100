# Contributing

欢迎改进这个开源 Python100 课程。

## 内容原则

- 难度要有承接性：不要在前置知识未出现时提前使用高级概念。
- 每一天都要有最小可运行案例。
- 示例必须使用虚拟数据，不提交真实身份、账号、密钥、交易账户或私人资料。
- 练习题要从简单到稍难，服务于数据分析、Quant、LLM 或 Agent 项目能力。
- Hot100 / LeetCode 挑战从 Day21 开始，前 20 天只服务零基础和工程习惯。
- 文风保持工程知识库风格：输入、处理、输出、失败情况、复盘方式要清楚。

## 提交流程

1. 修改对应 Day 文件或文档。
2. 运行 `python3 scripts/build_mobile_site.py` 更新网页。
3. 检查是否包含隐私信息或不可公开资料。
4. 提交 PR，并说明修改了哪些天数、解决什么问题。
