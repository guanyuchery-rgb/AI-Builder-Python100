# Python100 能力达标标准

这个文件回答一个问题：学完 Python100 后，到底算不算有比较强的 Python 能力。

结论先说清楚：达标不靠读完文本，而靠代码、运行结果、Debug 记录、IC 工程验证和可展示作品。

## 达标口径

学完 Day01-Day100，并持续完成 IC 验证后，目标能力是：

- 掌握 Python 主要语法和常用标准库。
- 能处理 CSV、JSON、SQLite、API 和 pandas 数据。
- 能把模糊任务拆成输入、输出、函数、测试、日志和 README。
- 入门 LLM、Agent、RAG 和 Quant 的核心概念。
- 能做 4 个轻量作品工程：数据分析、Quant、LLM、Agent。

## 最低证据

必须留下这些证据：

- Day 课程代码能运行。
- 每个阶段有 Debug 记录。
- IC 工程挑战有代码、README 和运行结果。
- 4 个最终工程有截图、运行说明和简历描述。
- 能口头解释每个工程的输入、输出、边界和改进方向。

## 不达标信号

- 只读文档，不手打代码。
- 只复制答案，不看报错。
- 不记录 Debug 过程。
- 工程不能重新运行。
- 说不清为什么用某个库或 API。

## 能力判断

如果能做到上面的最低证据，学完后可以达到“较强 Python 实战入门到进阶过渡”的水平。

更具体地说：不是算法竞赛能力，而是面向数据分析、AI 工具、Quant 研究和轻量工程交付的 Python 能力。
