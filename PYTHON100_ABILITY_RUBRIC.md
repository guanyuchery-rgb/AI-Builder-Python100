# Python100 能力达标标准

这个文件用来判断学完 100 天后是否真的形成 Python 实战能力。

结论先写清楚：只读完课程，不算达标。

达标必须有代码、运行结果、刷题记录、复盘和可展示作品。

## 目标定位

学完后应达到的是：

- 能独立写中小型 Python 数据工具。
- 能处理真实数据输入、错误、日志、测试和文档。
- 能用 pandas、SQL、SQLite、Streamlit、FastAPI 做轻量应用。
- 能理解并实现常见面试算法题的主流解法。
- 能把统计、金融、数据分析背景转成 Python 项目表达。

这不是资深工程师水平。

也不是只靠 100 天就稳拿工作的承诺。

但如果每天认真做，可以达到找数据分析、量化研究助理、AI 应用实习、Python 数据工具方向时有说服力的作品基础。

## 最低达标线

必须同时满足：

- Day01-Day100 的核心代码都手打并运行过。
- 每 10 天至少完成 1 个可复现小产物。
- 至少 60 道 LeetCode 官方 Accepted 记录；时间充足时逐步扩展到 100 道。
- 至少 20 道题写过二刷复盘。
- 至少 4 个项目过程文档能公开展示。
- 至少 1 个 Streamlit 或 FastAPI 应用能从 README 跑起来。
- 至少 1 个数据分析或 Quant 小项目能解释输入、指标、结果和限制。
- 至少 1 个 LLM/RAG/Agent 小工具能解释 prompt、schema、错误边界和评估方法。

## 分阶段验收

| 阶段 | 天数 | 达标证据 |
|---|---:|---|
| Python 基础 | Day01-Day20 | 能写函数、文件读写、异常处理、CLI、测试，不靠复制粘贴跑通 |
| 数据分析 | Day21-Day35 | 能读取、清洗、聚合、可视化数据，并输出 Markdown 报告 |
| 工程交付 | Day36-Day45 | 能写 API client、FastAPI、Streamlit、Docker/CI 最小说明 |
| LLM / Agent | Day46-Day60 | 能写结构化 prompt、schema 校验、RAG 检索和工具输入输出 |
| Quant | Day61-Day75 | 能计算收益率、回撤、信号、手续费，并保存回测结果 |
| 作品集 | Day76-Day100 | 能把项目过程、README、复盘和面试故事整理成公开材料 |

## 算法题达标线

LeetCode 不是为了炫技。

它用于证明你能写清楚状态、边界和复杂度。

最低建议：

- 哈希表：10 题。
- 双指针 / 滑动窗口：10 题。
- 栈 / 队列 / 堆：8 题。
- 二分：6 题。
- 链表：6 题。
- 树：8 题。
- 图：6 题。
- 动态规划：8 题。
- 回溯 / 贪心 / 矩阵：8 题。

课程内提供 720 个分层训练位，覆盖约 100 道独立官方题目。

总数可以超过 60，但少于 60 时，求职说服力会弱。

前期不要求硬做挑战题。先完成必做题，再逐步做选做题和挑战题。

## 项目达标线

每个可展示项目都要能回答：

- 输入是什么。
- 输出是什么。
- 核心函数在哪里。
- 如何运行。
- 如何测试。
- 出错时怎么看日志。
- 这个项目的限制是什么。
- 这次实现训练了什么能力。

项目不需要很大。

但必须能跑、能解释、能复现。

## 能力核算结论

如果学习者已经完成前 10 天，并且有统计、金融或数据背景：

- 只看文本：做不到较强 Python 能力。
- 跟着手打代码但不复盘：能到基础可用，但求职说服力不足。
- 手打代码、完成项目产物、LeetCode 有 60+ Accepted、能讲清项目过程：可以达到较强 Python 实战能力。
- 如果进一步做到 100 道左右 Accepted，并能复盘常见题型，会更接近求职面试的稳定准备状态。
- 想达到开发岗强竞争力：还需要继续补系统设计、数据库、后端工程、更多真实项目和面试训练。

这套 100 天路径的合理目标是：

把一个统计/金融/数据背景的人，从 Python 基础学习者，推到能用 Python 独立做数据工具、轻量应用、算法复盘和项目作品集的人。
