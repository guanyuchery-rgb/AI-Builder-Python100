# AI Builder Python100 学习内容重构原则

本课程不是语法学习路线，而是工程问题解决路线。

核心顺序从：

```text
语法 -> 定义 -> 例题
```

改为：

```text
问题 -> 场景 -> 为什么存在 -> 怎么解决 -> 语法实现
```

## 每个知识点统一结构

### 1. 是什么（What）

用不超过 3 句话解释概念，避免复杂术语，并给出一句最简定义。

### 2. 为什么存在（Why）

回答：如果没有这个东西，会出现什么问题。

### 3. 真实工程场景（Real Scenario）

语法讲解之前，先说明它会在哪些真实任务中出现：

- 科研场景
- 数据分析场景
- Quant 场景
- LLM/Agent 场景

### 4. 怎么做（How）

先给流程，再给代码：

```text
场景
  ↓
流程图
  ↓
代码
```

### 5. 工程设计思维（Design）

解释工程师为什么这样写，为什么不直接写，为什么要多这一层。

### 6. 项目中的位置（Where）

说明这个知识点未来会在哪些项目中再次出现，例如：

- Quant 回测框架
- FastAPI / Streamlit
- Agent tool
- CLI 工具
- 数据处理脚本
- GitHub 开源作品集

## 课程重构原则

禁止把新内容写成：

```text
定义 -> 语法 -> 例题
```

统一改为：

```text
真实任务
  ↓
拆解流程
  ↓
引出问题
  ↓
引出知识点
  ↓
解释设计原因
  ↓
代码实现
  ↓
未来项目应用
```

## 掌握标准

课程目标不是“学会 Python 语法”，而是学会把真实任务拆成函数、模块、流程和工程结构。

每个知识点最终必须能回答四个问题：

1. 它是什么？
2. 它为什么存在？
3. 它解决什么问题？
4. 它在未来 Quant / LLM / Agent 项目中怎么使用？

只有能够回答这四个问题，才算真正掌握。
