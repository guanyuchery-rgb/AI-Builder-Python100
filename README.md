## 学习原则

- 中文理解为主，必要英文术语保留。
- 每天约 2 小时，短讲解 + 题目驱动。
- 每天至少跑一段代码。
- 每天完成 5 道递进题，风格参考 Hot100/LeetCode，但场景偏工业界。
- 每题配参考链接、工业意义和带注释解法骨架。
- 每天记录一个 Debug 或可复用模式。
- 先形成稳定小工具，再扩展到 Streamlit、FastAPI、Agent。
