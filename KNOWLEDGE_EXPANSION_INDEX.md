# Python100 知识广度索引

这个索引用来回答一个问题：

> 前面基础阶段没有展开的知识点，应该在后面哪一天、哪个工业化场景里补？

原则：

- Day01-Day09 保持轻量，不回头塞太多内容。
- Day10 之后按工程场景补充知识广度。
- 每个知识点都要能连接到数据分析、Quant、LLM/Agent 或作品集项目。

## 学习使用方式

如果你已经学到 Day10，不建议回头把 Day01-Day09 重新塞满。更稳的节奏是：

1. Day10 以后遇到真实任务时，再补相关语法。
2. 每次只把语法放进一个工程问题里理解。
3. 每个新知识点都写清输入、处理、输出、失败点。
4. 能迁移到一个小项目，才算真正掌握。

这条路线的目标不是“背完整 Python 百科”，而是逐步形成自己的 AI / 数据 / Quant / Agent 工程知识库。
