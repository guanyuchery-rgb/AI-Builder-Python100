## 基础详细讲解

### 如果你完全看不懂，先只抓这一句话

今天学的是三种“装数据”的方式：**字符串装文本，字典装一条有字段的数据，集合装不重复的数据。**

```text
文本内容 -> 字符串 str
字段记录 -> 字典 dict
去重名单 -> 集合 set
```

不要一开始就背所有方法。先知道什么时候用谁：

- 看到一段话、一行输入、一个标签：先想字符串。
- 看到 `topic -> python`、`hours -> 2` 这种字段和值：先想字典。
- 看到“去重”“是否出现过”：先想集合。

### 三种结构怎么读

```python
text = " Python "
clean = text.strip().lower()
```

这段表示：拿到一段文本，先去掉两边空格，再转小写。

```python
record = {"topic": "python", "hours": 2}
print(record["topic"])
```

这段表示：一条记录有两个字段，`topic` 字段的值是 `python`。

```python
tags = {"python", "sql", "python"}
print(tags)
```

集合会自动去重，所以重复的 `python` 只保留一份。

### 数据流追踪

| 代码 | 发生了什么 |
| --- | --- |
| `text = " Python "` | 创建一段带空格的字符串 |
| `text.strip()` | 得到 `"Python"`，原字符串不变 |
| `.lower()` | 得到 `"python"` |
| `record["topic"]` | 从字典里按 key 取 value |
| `"python" in tags` | 判断集合里是否出现过这个值 |

### 最容易混的点

| 卡点 | 正确理解 |
| --- | --- |
| `list` 和 `dict` 都能装多个值 | list 靠位置，dict 靠字段名 |
| `record[0]` 取不到字典第一个字段 | 字典要用 key，例如 `record["topic"]` |
| `set` 不能按顺序取第几个 | set 用来去重和判断出现，不用来保存顺序 |
| `strip()` 会不会改原字符串 | 不会，它返回一个新字符串 |

### 字符串、字典与集合

字符串处理文本，字典处理 key-value 映射，集合处理去重和成员判断。三者是实际主题中最常见的数据结构：文件路径、字段名、配置项、用户输入、模型输出，很多都是文本和映射。

### 基础知识深讲：字符串、字典、集合的角色

字符串是不可变的文本序列。`strip()`、`lower()`、`split()` 看起来只是小工具，但它们解决的是文本清洗的基本问题：去掉多余空格、统一大小写、按规则拆分。以后处理日志、Prompt、CSV 字段、网页文本时，这些操作会反复出现。

字典是 key-value 映射。它适合表达“字段名 -> 字段值”，比如 `{"topic": "python", "minutes": 120}`。字典比列表更有语义，因为 `record["minutes"]` 明确说明你取的是分钟数，而 `record[1]` 只能靠记位置。结构化数据、JSON、配置文件、LLM 输出 schema 都离不开字典。

这三类结构的选择标准很简单：有顺序的一组数据用 list，有字段含义的一条记录用 dict，需要去重或判断是否出现过用 set。选对结构，代码会自然变清楚；选错结构，后面逻辑会越来越绕。

### 手写顺序

- 字符串先练 `strip()`、`lower()`、`split()`，它们能解决大量清洗问题。
- 字典先练通过 key 取值和更新，例如 `record["score"]`。
- 集合先练去重和判断是否出现过，例如 `x in seen`。

讲解：为后面数据清洗做准备。
