## 知识点 1：测试 Test

### 定义

测试是用可重复方式验证代码行为是否符合预期。

### 为什么存在

学习阶段靠眼睛看输出可以，但项目阶段必须能重复验证，否则改动会破坏旧功能。

### 最小案例

```python
def simple_return(start_price: float, end_price: float) -> float:
    return (end_price - start_price) / start_price


def test_simple_return() -> None:
    assert simple_return(100, 110) == 0.10
    assert simple_return(100, 90) == -0.10


test_simple_return()
print("tests passed")
```

### 常见错误

- 只测试正常情况。
- 浮点数直接精确比较。
- 测试依赖本地随机文件。

### 工程应用

- 指标函数测试。
- 数据清洗规则测试。
- Agent tool 输入输出测试。
- API client 测试。

### 未来扩展

- pytest。
- fixture。
- CI。
- mock。

## 知识点 2：日志 Logging

### 定义

日志是程序运行时记录状态、事件和错误的方式。

### 为什么存在

项目出错时，需要知道发生了什么、输入是什么、失败在哪里。

### 最小案例

```python
import logging

logging.basicConfig(level=logging.INFO, format="%(levelname)s:%(message)s")


def process_symbol(symbol: str) -> str:
    logging.info("processing symbol=%s", symbol)
    return symbol.strip().upper()


print(process_symbol(" aapl "))
```

### 常见错误

- 用 print 到处调试但不删除。
- 日志没有上下文。
- 把密钥或隐私数据写进日志。

### 工程应用

- 数据管道运行记录。
- 回测过程记录。
- Agent tool 调用轨迹。
- API 请求排错。

### 未来扩展

- 日志文件。
- structured logging。
- tracing。

## 面试角度 Interview

能说明测试验证结果，日志记录过程，Debug 追踪根因。三者不是一回事。

## Quant 关联

量化指标函数必须测试，否则回测结果不可信。日志能记录实验参数和运行状态。

## LLM / Agent 关联

Agent 输出必须可验证。工具调用日志能帮助判断 Agent 是计划错、输入错，还是工具错。

#### 2. Easy - unittest 表格测试

题目：测试多个输入输出组合。

            ((0.1, 0.4, -0.05), "HIGH"),
        ]
        for args, expected in cases:
            self.assertEqual(risk_level(*args), expected)

#### 3. Medium - logging 配置

题目：记录任务开始、成功、失败。

讲解：print 适合学习，logging 适合项目。

```python
import logging

logging.basicConfig(level=logging.INFO, format="%(levelname)s:%(message)s")
logger = logging.getLogger(__name__)

def run_job() -> None:
    logger.info("job started")
    try:
        logger.info("job finished")
    except Exception:
        logger.exception("job failed")
        raise
```

#### 4. Medium - 最小复现

题目：把一个 KeyError 缩成最小案例。

讲解：Debug 时先缩小问题，再改代码。

```python
# 原因: data 没有 price 字段
# 修复: 用 get 或先校验字段
def get_price(data: dict) -> float | None:
    return data.get("price")
```

#### 5. Hard - Fake Client 测试 Agent Tool

题目：不用真实 API，测试 tool 逻辑。

讲解：测试要控制外部依赖，避免网络影响结果。

```python
class FakeClient:
    def quote(self, symbol: str) -> dict:
        return {"symbol": symbol, "price": 100.0}

def quote_tool(client, symbol: str) -> dict:
    if not symbol:
        return {"ok": False, "error": "symbol required"}
    return {"ok": True, "data": client.quote(symbol)}

def test_quote_tool():
    result = quote_tool(FakeClient(), "SPY")
    assert result["ok"] is True
    assert result["data"]["price"] == 100.0
```
