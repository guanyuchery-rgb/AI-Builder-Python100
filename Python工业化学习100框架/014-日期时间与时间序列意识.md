## 基础详细讲解

### 如果你完全看不懂，先只抓这一句话

今天学的是：**日期不是普通字符串，时间数据要先解析，再排序、比较、计算。**

```text
"2026-06-01" -> datetime 对象 -> 可以排序/比较/计算间隔
```

### 为什么不能只当字符串

字符串只是文本。日期对象才有时间含义。

```python
from datetime import datetime

day = datetime.strptime("2026-06-01", "%Y-%m-%d")
print(day)
```

`strptime` 可以理解成：按指定格式把字符串解析成日期。

### 格式必须对上

| 字符串 | 格式 |
| --- | --- |
| `2026-06-01` | `%Y-%m-%d` |
| `2026/06/01` | `%Y/%m/%d` |
| `06-01-2026` | `%m-%d-%Y` |

格式不匹配就会报错。

### 时间序列最重要的三件事

| 检查 | 为什么 |
| --- | --- |
| 是否解析成日期 | 否则排序可能错 |
| 是否按时间排序 | 否则收益率和窗口计算会错 |
| 区间边界是否清楚 | 否则实验不可复现 |

### Debug 对照表

| 现象 | 常见原因 | 怎么改 |
| --- | --- | --- |
| `ValueError` | 日期格式和模板不匹配 | 检查 `%Y-%m-%d` 等格式 |
| 排序奇怪 | 还在按字符串排序 | 先解析成 datetime |
| 少一天/多一天 | 边界是否包含没写清楚 | 明确 start/end 是否包含 |

### 日期时间与时间序列意识

时间不是普通字符串。日期需要解析、比较、排序和对齐。统计和 Quant 尤其依赖时间边界：今天、昨天、滚动窗口、交易日、样本区间，都必须清楚。

### 基础知识深讲：时间对象与样本区间

日期字符串只是文本，日期对象才可以稳定比较、排序和计算差值。`"2026-10-01"` 和 `"2026-2-01"` 如果按字符串比较可能出问题；解析成 datetime 后，Python 才知道它们是真正的时间点。

时间序列最重要的是顺序和边界。顺序错了，移动平均、收益率、回撤都会错；边界不清，样本区间就无法复现。你以后做 Quant 时，必须能写清楚实验从哪一天开始，到哪一天结束，是否包含当天。

时间处理还要区分日期、时间、时区和交易日。基础阶段先掌握日期解析和排序；后面再处理时区、节假日、交易日历。不要一开始把所有复杂时间问题塞进来，但要知道它们是未来风险点。

时间字段进入项目后，第一步应检查格式一致性和缺失值。一个混有 `2026-06-01`、`06/01/2026`、空字符串的日期列，必须先清洗再参与计算。

### 手写顺序

- 用 `datetime.strptime()` 把字符串转成日期对象。
- 排序前确认字段真的已经被解析成日期。
- 写清时间范围的开始和结束，避免边界不一致。

### 今天必须真正弄懂

- **输入**：今天这段代码从哪里拿到数据，是手写值、终端输入、文件，还是函数参数。
- **处理**：每一行代码改变了什么状态，例如变量变了、列表多了一个元素、判断走了某个分支。
- **输出**：结果给谁看，是终端输出、返回值、文件，还是后面函数继续使用。
