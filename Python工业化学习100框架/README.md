# Python工业化学习100框架

这是面向 LLM、Agent、量化学习和数据分析的 Python 工业化学习框架。

当前版本覆盖 Day01-Day100：前 20 天打基础，Day21 做项目衔接，Day22-Day60 进入数据工程、轻量应用、LLM/Agent 和 Quant 项目化训练，Day61-Day100 进入项目实战、作品集和长期维护。

命名和内容采用“中文为主 + 必要英文术语”的方式：中文帮助理解，英文保留工程、搜索、面试和文档阅读需要的关键词。

## 零基础递进规则

- Day01：只写 `print()`、注释、字符串和最简单数字计算。
- Day02：只新增变量、类型转换、`input()` 和 f-string。
- Day03：才开始写 `if / elif / else`。
- Day04：才开始写 `for`、`range()` 和计数器。
- Day05：才开始写列表和最小算法。
- Day06：才开始写函数和 `main()`。
- Day07-Day20：在前面能力稳定后，再逐步进入数据清洗、对象、文件、API 和小项目。
- Day21-Day100：继续按小步递进，不突然跳到大项目；每天只新增一个工程能力，并配套 7 道简单题和 5 道巩固题。

## 每天 2 小时学习节奏

- 15 分钟：读当天目标和最小代码。
- 25 分钟：手打一遍核心示例。
- 35 分钟：完成 7 道简单路线题。
- 35 分钟：完成 5 道递进题。
- 10 分钟：记录 Debug、边界情况和项目迁移点。

## 阶段划分

- Day01-Day20：Python 工业化基础。
- Day21：6 月项目衔接路线。
- Day22-Day32：数据工程、pandas、NumPy、SQLite。
- Day33-Day40：配置、CLI、日志、测试、FastAPI、Streamlit、CI。
- Day41-Day50：LLM、RAG、Agent tool、评估、安全边界。
- Day51-Day60：Quant 数据、指标、回测、报告和 Capstone。
- Day61-Day75：三个项目原型：Quant、LLM、Agent。
- Day76-Day90：应用化、服务化、RAG 和策略项目。
- Day91-Day100：Agent 演示、作品集、复盘和长期维护。

## 学习原则

- 先能运行，再追求优雅。
- 前 6 天不提前写项目，先按顺序打牢基础。
- Day06 之后先写小函数，再写大项目。
- 先 CLI，再 Streamlit/FastAPI。
- 先手写理解，再让 AI review。
- 遇到重复错误，必须沉淀到 Debug。
- 每天只追求一个稳定的小进步。

## 学完后的方向

Day01-Day100 完成后，保留三个长期项目：Quant Research Starter、LLM Report Assistant、Agent Tool Prototype。
