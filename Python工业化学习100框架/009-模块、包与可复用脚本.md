## 知识点结构

### 定义

把代码拆成模块，避免所有逻辑写在一个文件，是把一个具体问题拆成可运行、可检查、可复用代码的过程。

### 为什么存在

它让学习从临时笔记变成长期资产。公开仓库里的每一天都应该能让读者看到：今天学什么、为什么学、怎么运行、如何检查结果。

### 最小案例

```python
def build_message(topic):
    return f"今天学习：{topic}"

def main():
    print(build_message("module basics"))

if __name__ == "__main__":
    main()
```

### 常见错误

- 文件一被 import 就自动运行大量逻辑。
- 函数名不能表达用途。
- 忘记 `if __name__ == "__main__"` 的入口保护。

## 基础详细讲解

### 模块、包与可复用脚本

模块就是一个 `.py` 文件，包是一组模块。把代码拆成模块，是为了让项目从“一个长脚本”变成“多个职责清楚的小文件”。`import` 不是炫技，而是复用。

### 手写顺序

- 把计算函数放进 `utils.py`，把运行入口放进 `main.py`。
- 用 `if __name__ == "__main__":` 区分被导入和直接运行。
- 每个文件只承担一个主要职责，先少拆，再逐步拆。

### 今天必须真正弄懂

- **输入**：今天这段代码从哪里拿到数据，是手写值、终端输入、文件，还是函数参数。
- **处理**：每一行代码改变了什么状态，例如变量变了、列表多了一个元素、判断走了某个分支。
- **输出**：结果给谁看，是终端输出、返回值、文件，还是后面函数继续使用。

### 和后面项目的关系

后面 FastAPI、Streamlit、Quant backtest、Agent tool 都会依赖清晰模块结构。

## 5 道基础巩固题

#### 1. 消息函数

题目：写函数返回学习消息。

讲解：模块里先放小函数。

```python
def build_message(topic):
    return f"今天学习：{topic}"

print(build_message("module"))
```

#### 2. main 函数

题目：用 main 管理入口。

讲解：项目脚本常见结构。

```python
def main():
    print("run main")

main()
```

#### 3. 入口保护

题目：添加 `__main__` 判断。

讲解：方便未来被 import。

```python
def main():
    print("safe run")

if __name__ == "__main__":
    main()
```

#### 4. 拆分处理逻辑

题目：一个函数做格式化，一个函数做输出。

讲解：避免所有逻辑混在一起。

```python
def format_topic(topic):
    return topic.strip().title()

print(format_topic(" python "))
```

#### 5. 写模块说明

题目：用注释说明模块用途。

讲解：开源代码要能被别人读懂。

```python
# day09_module.py
# 作用：演示 main 入口
print("ok")
```
