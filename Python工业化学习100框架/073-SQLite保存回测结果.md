### 带注释参考代码：Number of Islands

```python
def num_islands(grid):
    rows, cols = len(grid), len(grid[0])

    def dfs(r, c):
        if r < 0 or r >= rows or c < 0 or c >= cols:
            return
        if grid[r][c] != "1":
            return
        grid[r][c] = "0"
        dfs(r + 1, c)
        dfs(r - 1, c)
        dfs(r, c + 1)
        dfs(r, c - 1)

    count = 0
    for r in range(rows):
        for c in range(cols):
            if grid[r][c] == "1":
                count += 1
                dfs(r, c)
    return count
```

## 强化练习与工程使用

> 所属阶段：Quant 研究与回测基础。目标是：把行情样例变成可复查的研究实验。

### 工程使用场景

今天的主题是 **SQLite 保存回测结果**，不要只把它当成一个孤立知识点。它应该落到 `Quant Research Starter` 里的一个可复用能力：输入清楚、处理过程可检查、输出可以保存，失败时能解释原因。重点检查：未来函数、样本区间、交易成本、风险指标。

### 强化 7 题

1. **复现题**：不看答案重写今天最小案例，并写出运行命令和预期输出。
2. **边界题**：构造 3 个异常输入，说明程序应该返回结果、跳过、还是报错。
3. **数据题**：准备 5-10 条虚拟样例数据，覆盖正常值、空值、重复值和极端值。
4. **函数题**：把核心逻辑封装成 1 个函数，函数名、参数名和返回值必须能读懂。
5. **测试题**：写 2 个正常测试和 1 个边界测试，确认修改后行为不变。
6. **算法题**：完成 [Top K Frequent Elements](https://leetcode.com/problems/top-k-frequent-elements/)，题型：堆 / TopK。提示：先计数再取 TopK，连接日志分析和检索排序。
7. **项目题**：把今天结果接入 `Quant Research Starter` 的一个小模块，并在 README 里写 3 行使用说明。

### 难度控制

Medium 到 Hard 过渡，重点是滑动窗口、前缀和、DP 和边界条件。 把题目连接到收益、回撤、仓位、成本、样本区间和实验记录。 如果算法题暂时写不出完整代码，先写暴力思路、输入输出样例和复杂度，再补优化版。
