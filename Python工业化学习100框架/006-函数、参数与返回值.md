### 最小案例

```python
def average(values):
    if not values:
```

## 基础详细讲解

### 函数、参数与返回值

函数是把一段可重复使用的逻辑装进一个名字。参数是输入，`return` 是输出。初学时不要追求复杂函数，先保证一个函数只解决一个小问题，并且名字能说明它做什么。

### 基础知识深讲：函数契约与复用

函数的本质是一份契约：调用者提供什么输入，函数承诺返回什么输出。`def average(values):` 这行代码定义了函数名和输入入口；`return` 定义输出出口。只要契约清楚，函数就可以被脚本、测试、CLI、API 或 Agent tool 反复调用。

`print()` 和 `return` 的差别必须吃透。`print()` 是把信息显示给人看，返回值则是把结果交给程序继续使用。一个只打印不返回的函数，很难参与后续计算；一个有返回值的函数，可以被测试、组合和复用。工程里核心逻辑通常应该 return，入口层再负责 print 或保存文件。

函数参数应该尽量减少对外部变量的依赖。比如 `average(values)` 比直接使用全局 `prices` 更稳，因为它只依赖传入的 values。这样你可以用不同列表测试它，也可以把它放进别的项目。这个习惯会直接影响后面的 pytest、API 参数、Agent tool schema。

边界输入是函数深度理解的一部分。空列表、负数、None、字符串数字，都可能进入函数。初学不需要一次处理所有情况，但必须养成习惯：每写一个函数，都问一句“最小正常输入是什么，最危险的边界输入是什么”。

### 手写顺序

- 先写不用函数的版本，确认逻辑跑通；再把稳定逻辑提取成函数。
- 函数内部少用外部变量，尽量通过参数传入数据。
- 区分 `print()` 和 `return`：前者给人看，后者把结果交给后续代码。

### 零基础拆解：函数输入输出契约

#### 这一天到底在练什么

函数把一段逻辑变成可复用工具。参数是输入，return 是输出。你写函数时先不要想高级语法，只问：这个函数要什么数据，返回什么结果。

#### 手写时按这个顺序

- 先写不用函数的平均值计算。
- 把输入变成参数 `values`。
- 把最终结果从 `print()` 改成 `return`，再在外面打印函数调用结果。

#### 最常见卡点

函数里只 `print` 会导致后面拿不到结果；忘记调用函数时，程序不会自动执行函数体。

- 我能给空列表返回一个合理结果。

## 5 道基础巩固题

#### 1. 加法函数

题目：写一个返回两数之和的函数。

讲解：函数要 return。

```python
def add(a, b):
    return a + b

print(add(2, 3))
```

#### 2. 平均值函数

题目：封装平均值计算。

讲解：处理空列表。

```python
def average(values):
    if not values:
        return 0
    return sum(values) / len(values)

print(average([1, 2, 3]))
```

#### 3. 收益率函数

题目：计算开始价格到结束价格的收益率。

讲解：Quant 常用函数。

```python
def simple_return(start, end):
    return end / start - 1

print(simple_return(100, 105))
```

#### 4. 价格校验函数

题目：判断价格是否大于 0。

讲解：为数据校验做准备。

```python
def is_valid_price(price):
    return price > 0

print(is_valid_price(100))
```

#### 5. main 入口

题目：用 main 管理脚本入口。

讲解：后面 CLI 会用到。

```python
def main():
    print("run")

if __name__ == "__main__":
    main()
```
