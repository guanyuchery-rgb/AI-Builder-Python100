## 核心讲解

今天的关键词是 **把重复逻辑封装成可复用函数**。函数不是一开始就为了“高级工程”，而是为了解决一个很朴素的问题：同一段逻辑反复写，很容易写错、改漏、看不懂。

先看一个不用函数的例子：

```python
prices1 = [100, 102, 105]
avg1 = sum(prices1) / len(prices1)
print(avg1)

prices2 = [98, 99, 101]
avg2 = sum(prices2) / len(prices2)
print(avg2)
```

这里平均值逻辑写了两次。如果以后你想处理空列表，就要改两处；如果有十组价格，就要写十次。函数就是把这段稳定逻辑收起来，给它一个名字，以后需要时直接调用。

```python
def average(values):
    if not values:
        return 0
    return sum(values) / len(values)

print(average([100, 102, 105]))
print(average([98, 99, 101]))
```

今天最重要的不是背 `def`，而是看懂这四件事：

- **定义函数**：告诉 Python “以后有一个叫 average 的工具”。
- **调用函数**：真正让这个工具开始执行。
- **参数**：调用时交给函数的数据。
- **返回值**：函数处理完后交回来的结果。

从工程角度看，函数是所有项目的基本模块。数据清洗函数、收益率函数、Agent tool、API handler、测试用例，都是围绕“输入 -> 处理 -> 输出”组织的。

## 知识点结构

### 定义

函数是一个有名字的小工具。它把一段逻辑包装起来，等你调用它时再执行。

```python
def 函数名(参数):
    函数体
    return 结果
```

这四个部分要分清：

- `def`：告诉 Python 下面要定义函数。
- `函数名`：以后调用这个工具时用的名字。
- `参数`：函数需要的输入。
- `return`：函数交回来的输出。

### 为什么存在

函数存在，是为了减少重复、降低错误、提高复用。

不用函数时，你经常会把同一段逻辑复制很多遍。复制越多，未来修改越危险。用函数后，逻辑只写一次，其他地方只负责调用。以后要修边界、改公式、加日志，只改函数内部即可。

### 最小案例

```python
def add(a, b):
    result = a + b
    return result

answer = add(2, 3)
print(answer)
```

执行顺序不是从函数里面开始，而是：

1. Python 看到 `def add(a, b):`，记住有一个叫 `add` 的函数，但暂时不执行函数体。
2. Python 执行 `answer = add(2, 3)`，这才开始调用函数。
3. 调用时，`a` 绑定到 `2`，`b` 绑定到 `3`。
4. 函数内部计算 `result = a + b`。
5. `return result` 把 `5` 交回调用位置。
6. `answer` 得到 `5`，最后 `print(answer)` 输出。

### 常见错误

- 函数里只 `print` 不 `return`，导致结果不能复用。
- 只定义函数，没有调用函数，以为函数会自动执行。
- 参数名含义不清楚。
- 调用时传入的参数数量不对。
- 把所有代码都塞进一个巨大函数。

## 基础详细讲解

### 如果你完全看不懂，先只抓这一句话

函数就是：**把一段代码起个名字，等需要时把数据交给它，让它算完后把结果还回来。**

先不要背“封装”“抽象”“模块化”这些词。Day06 只抓住一条线：

```text
我给函数数据 -> 函数内部计算 -> 函数把结果还给我
```

比如：

```python
def add(a, b):
    return a + b

answer = add(2, 3)
print(answer)
```

这段代码只做一件事：

```text
我给 add 两个数 2 和 3 -> add 内部做加法 -> add 还给我 5
```

你读函数时永远按这个顺序看：

1. 先找函数名：这个工具叫什么。
2. 再看参数：这个工具需要什么输入。
3. 再看函数体：这个工具内部做什么。
4. 最后看 return：这个工具把什么结果还回来。

### 函数不是数学里的“高深函数”

很多人卡在“函数”这个词，是因为它听起来像数学。Python 里的函数更像一个小机器：

```text
          输入
           |
           v
    +---------------+
    |   函数内部     |
    |   处理逻辑     |
    +---------------+
           |
           v
          输出
```

比如榨汁机：

```text
苹果 -> 榨汁机 -> 苹果汁
```

Python 函数：

```text
[1, 2, 3] -> average -> 2
```

函数名就是机器名，参数就是放进去的原料，`return` 就是吐出来的结果。

### 函数、参数与返回值

函数是把一段可重复使用的逻辑装进一个名字。参数是输入，`return` 是输出。初学时不要追求复杂函数，先保证一个函数只解决一个小问题，并且名字能说明它做什么。

你可以先把函数理解成一个“处理器”：

```text
输入数据 -> 函数内部处理 -> 返回结果
```

比如平均值函数：

```text
[1, 2, 3] -> average -> 2
```

如果你不能用这条线解释一个函数，说明这个函数还没有想清楚。

### 逐行执行追踪：一定要按这个表读

代码：

```python
def add(a, b):
    result = a + b
    return result

answer = add(2, 3)
print(answer)
```

执行追踪：

| 顺序 | Python 看到的代码 | 真正发生的事 |
| --- | --- | --- |
| 1 | `def add(a, b):` | Python 记住一个叫 `add` 的函数，函数体暂时不执行 |
| 2 | `answer = add(2, 3)` | 这里出现了 `add(...)`，函数开始被调用 |
| 3 | `a = 2, b = 3` | 调用时，参数 `a` 得到 2，参数 `b` 得到 3 |
| 4 | `result = a + b` | 函数内部计算，`result` 变成 5 |
| 5 | `return result` | 函数把 5 还给调用位置 |
| 6 | `answer = add(2, 3)` | `add(2, 3)` 整体变成 5，所以 `answer = 5` |
| 7 | `print(answer)` | 输出 5 |

最关键的一句是：**`add(2, 3)` 调用结束后，会被返回值替换掉。**

你可以把这行：

```python
answer = add(2, 3)
```

在脑子里替换成：

```python
answer = 5
```

如果能理解这一步，函数就通了一半。

### 参数绑定图：a 和 b 什么时候有值

定义函数时：

```python
def add(a, b):
    return a + b
```

这时 `a` 和 `b` 还没有具体值。它们只是两个“入口名字”。

调用函数时：

```python
add(2, 3)
```

才发生绑定：

```text
a <- 2
b <- 3
```

再调用：

```python
add(10, 20)
```

这次绑定变成：

```text
a <- 10
b <- 20
```

所以 `a`、`b` 不是固定数字。它们像临时标签，每次调用函数时贴到新的输入数据上。

### return 回填图：结果回到哪里

看这段：

```python
def double(x):
    return x * 2

value = double(5)
print(value + 1)
```

执行时可以这样想：

```text
double(5)
=> return 5 * 2
=> return 10
```

所以：

```python
value = double(5)
```

等价于：

```python
value = 10
```

最后：

```python
print(value + 1)
```

就是：

```python
print(10 + 1)
```

输出 `11`。

### 为什么你会觉得函数难

函数难，不是因为语法长，而是因为它会让代码“跳来跳去”：

```text
先看到 def，但不执行
看到调用，跳进函数
遇到 return，跳回调用位置
继续执行后面的代码
```

所以读函数不能像读普通脚本那样只从上到下硬读。正确读法是：

```text
先跳过函数体 -> 找调用位置 -> 带着参数进入函数 -> 遇到 return 回来
```

这也是今天最重要的阅读能力。

### 基础知识深讲：函数契约与复用

#### 1. 函数先解决重复代码

先不要急着写 `def`。先看没有函数时为什么麻烦。

```python
values = [1, 2, 3]
total = sum(values)
count = len(values)
average_value = total / count
print(average_value)
```

这段代码只会算 `values` 这一组数据。如果你换成另一组数据，要重新写一遍。函数就是把“计算平均值”这件事抽出来：

```python
def average(values):
    total = sum(values)
    count = len(values)
    average_value = total / count
    return average_value
```

现在你可以给它不同输入：

```python
print(average([1, 2, 3]))
print(average([10, 20, 30]))
```

所以函数不是凭空出现的语法，而是从重复代码里抽出来的稳定逻辑。

#### 2. 定义函数不等于执行函数

这是 Day06 最容易卡住的地方。

```python
def say_hello():
    print("hello")
```

这段代码运行后不会输出 `hello`。原因是：`def` 只是定义函数，Python 只是记住“有一个叫 `say_hello` 的函数”。真正执行要靠调用：

```python
say_hello()
```

带括号才是调用。你可以把它理解成：

```text
def say_hello():  准备工具
say_hello()       使用工具
```

#### 3. 参数是函数入口，不是固定值

看这个函数：

```python
def add(a, b):
    return a + b
```

`a` 和 `b` 在定义函数时还没有具体值，它们只是占位名。调用时才会拿到值：

```python
add(2, 3)
```

这一刻发生的是：

```text
a = 2
b = 3
return a + b
```

再调用一次：

```python
add(10, 20)
```

这一刻又变成：

```text
a = 10
b = 20
return a + b
```

所以参数让同一个函数可以处理不同数据。

#### 4. return 是把结果带回调用位置

`return` 做两件事：

1. 结束函数执行。
2. 把结果交回函数被调用的地方。

```python
def add(a, b):
    return a + b

answer = add(2, 3)
```

这里 `add(2, 3)` 最后会变成 `5`，所以可以理解成：

```python
answer = 5
```

如果函数没有 `return`，Python 默认返回 `None`。

```python
def add_and_print(a, b):
    print(a + b)

answer = add_and_print(2, 3)
print(answer)
```

第一行输出 `5`，第二行输出 `None`。这就是为什么只会 `print` 不会 `return`，后面写项目会卡住。

#### 5. print 和 return 的区别

| 对比 | print | return |
| --- | --- | --- |
| 作用 | 给人看 | 给程序继续用 |
| 出现位置 | 函数内外都可以 | 只能在函数里有意义 |
| 后续能不能继续计算 | 不能直接用 | 可以赋值、判断、保存、测试 |
| 工程习惯 | 放在入口层、调试层 | 放在核心逻辑层 |

初学阶段可以这样记：

```text
想看结果：print
想把结果交给下一步：return
```

#### 6. 函数内部变量只在函数内部工作

```python
def add(a, b):
    result = a + b
    return result

print(result)
```

这段代码会报错，因为 `result` 是函数内部变量。它只在函数执行时临时存在，函数结束后外面不能直接访问它。

正确做法是接住返回值：

```python
answer = add(2, 3)
print(answer)
```

这也是为什么 `return` 很重要：它把函数内部结果带到函数外部。

原版 Python100 的函数章节会继续讲默认参数、关键字参数、不定长参数、匿名函数、变量作用域和模块。那些都重要，但你现在 Day06 最先要吃透的是：

- 会定义一个函数。
- 会调用一个函数。
- 知道参数什么时候有值。
- 知道 `return` 把值带回哪里。
- 会用返回值继续计算。

这些理解稳了，再学默认参数、模块和测试才不会混。

### 手写顺序

1. 先写不用函数的版本，确认逻辑能跑。
2. 找出重复逻辑，给它起一个清楚的名字。
3. 把会变化的数据放进参数。
4. 在函数内部完成计算。
5. 用 `return` 返回结果。
6. 在函数外调用函数，并用变量接住返回值。
7. 最后再 `print()` 这个变量，确认结果。

### 今天必须真正弄懂

- **定义函数时不会执行函数体**：只有调用时才执行。
- **参数是入口**：调用时传入的数据会绑定到参数名。
- **return 是出口**：函数的结果从这里回到调用位置。
- **print 不是 return**：看到结果不等于程序拿到了结果。

### 零基础拆解：函数输入输出契约

#### 这一天到底在练什么

今天只练一个能力：把“能跑的一段代码”变成“可以反复调用的小工具”。你写函数时先不要想高级语法，只问：这个函数要什么数据，内部做什么，返回什么结果。

#### 手写时按这个顺序

```python
# 第一步：不用函数，先把逻辑跑通
values = [1, 2, 3]
total = sum(values)
count = len(values)
answer = total / count
print(answer)
```

```python
# 第二步：把会变化的数据变成参数
def average(values):
    total = sum(values)
    count = len(values)
    answer = total / count
    return answer
```

```python
# 第三步：调用函数，并接住返回值
result = average([1, 2, 3])
print(result)
```

```python
# 第四步：处理边界输入
def average(values):
    if not values:
        return 0
    return sum(values) / len(values)
```

#### 最常见卡点

- 只写了 `def`，没有调用，所以没有任何输出。
- 函数里写了 `print()`，但没有 `return`，所以外面接到的是 `None`。
- 参数个数不一致，例如 `add(a, b)` 需要两个输入，但只传了一个。
- 把函数内部变量拿到外面直接用，导致 `NameError`。
- 函数名太模糊，例如 `do()`、`calc()`，以后复习不知道它算什么。

#### 函数 Debug 对照表

| 现象 | 常见原因 | 怎么改 |
| --- | --- | --- |
| 运行后没有任何输出 | 只定义了函数，没有调用 | 在函数外写 `函数名()` |
| 输出了 `None` | 函数内部只有 `print`，没有 `return` | 把结果 `return` 出来，再在外面 `print` |
| `TypeError: missing ... argument` | 调用时参数少了 | 检查 `def` 里需要几个参数，调用时传几个值 |
| `TypeError: takes 2 positional arguments but 3 were given` | 调用时参数多了 | 删除多余参数，或修改函数定义 |
| `NameError: result is not defined` | 在函数外访问函数内部变量 | 用 `answer = 函数名(...)` 接住返回值 |
| 结果不变 | 参数写死在函数内部，没有使用传入值 | 检查函数体是否真的用了参数 |

#### 三个必须亲手观察的实验

实验 1：只定义，不调用。

```python
def hello():
    print("hello")
```

运行后没有输出，这是正常的。因为函数还没被调用。

实验 2：调用函数。

```python
def hello():
    print("hello")

hello()
```

这次会输出 `hello`。

实验 3：print 不等于 return。

```python
def add(a, b):
    print(a + b)

answer = add(2, 3)
print(answer)
```

输出是：

```text
5
None
```

你要亲眼看到这个 `None`，才会真正记住：`print` 只是显示，`return` 才是交回结果。

## 7 道函数巩固题（从易到难）

#### 1. 加法函数

题目：写一个返回两数之和的函数。

讲解：先用最简单的 `add` 看懂参数和返回值。`a`、`b` 是参数名，调用时才会获得具体值。

```python
def add(a, b):
    result = a + b
    return result

answer = add(2, 3)
print(answer)
```

#### 2. 平均值函数

题目：封装平均值计算。

讲解：先判断空列表，再计算平均值。`return` 后面的值会回到调用位置。

```python
def average(values):
    if not values:
        return 0
    total = sum(values)
    count = len(values)
    return total / count

result = average([1, 2, 3])
print(result)
```

#### 3. 收益率函数

题目：计算开始价格到结束价格的收益率。

讲解：这是 Quant 里最小的函数模型。输入是开始价格和结束价格，输出是收益率。

```python
def simple_return(start, end):
    if start <= 0:
        return 0
    return end / start - 1

ret = simple_return(100, 105)
print(ret)
```

#### 4. 价格校验函数

题目：判断价格是否大于 0。

讲解：函数也可以返回 True / False。这样的函数常用于输入验证。

```python
def is_valid_price(price):
    return price > 0

ok = is_valid_price(100)
print(ok)
```

#### 5. print 和 return 对比

题目：观察只 print 不 return 的函数会发生什么。

讲解：这个题非常关键。你会看到函数内部打印了结果，但外部变量拿到的是 `None`。

```python
def add_and_print(a, b):
    print(a + b)

answer = add_and_print(2, 3)
print(answer)
```

#### 6. main 入口

题目：用 main 管理脚本入口。

讲解：`main()` 是项目入口函数。以后 CLI、脚本、测试都会更清楚。

```python
def main():
    result = add(2, 3)
    print(result)

def add(a, b):
    return a + b

if __name__ == "__main__":
    main()
```

#### 7. 函数阅读题

题目：读下面代码，写出执行顺序和最终输出。

讲解：读函数时，先跳过函数体，看哪里调用；调用时再进入函数。

```python
def double(x):
    return x * 2

value = double(5)
print(value + 1)
```
