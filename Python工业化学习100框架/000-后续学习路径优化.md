这个文件写进 Python100 课程主线内部。

它服务的学习状态是：


## 当前水平判断

已经具备：

- 能在终端创建目录、进入目录、运行 Python 文件。
- 能用编辑器改代码。
- 能写变量、循环、条件判断、函数。
- 接触过文件读写、异常处理、`main()` 入口。
- 能把一个数学规律写成多种 Python 解法。

主要短板：

- 路径和当前目录容易混。
- 文件名容易拼错。
- 有时会运行错文件。
- Debug 主要靠反复试，定位过程还不系统。
- 代码能跑，但复盘、测试、说明还不稳定。

所以后续学习原则是：

```text
先稳路径。
再稳函数。
再稳数据。
再稳复盘。
最后再追求项目复杂度。
```

## 创新思维训练

这里的创新不是做很大的东西。

而是把统计、数学、金融里的直觉，翻译成代码规则。

你已经出现过这种苗头：

```text
从假设检验、吸附壁、示性函数出发，
把抽象判断转成 Python 里的条件、符号、权重和算法优化。
```

这条路线要保留。

它说明学习不是只背语法，而是在形成自己的算法表达。

## 假设检验式思维

写代码时可以这样迁移：

```text
原假设：当前数据没有异常。
备择假设：当前数据存在异常。
证据：缺失率、波动率、阈值、错误次数。
决策：通过 / 拒绝 / 需要人工检查。
```

对应代码：

```python
def check_anomaly(value, threshold):
    if value > threshold:
        return "reject"
    return "pass"
```

可迁移到：

- 数据质量检查。
- Quant 风险阈值。
- LLM 输出可信度。
- Agent 是否需要人工审查。

## 吸附壁 / 阈值思维

“吸附壁”可以理解成边界或阈值。

在代码里，它常常变成：

- 上限。
- 下限。
- 停止条件。
- 风险边界。
- 自动拦截线。

示例：

```python
def clamp_score(score):
    if score < 0:
        return 0
    if score > 100:
        return 100
    return score
```

可迁移到：

- 数据清洗里的异常截断。
- Quant 仓位上限。
- Agent 工具调用次数上限。
- LLM 输出置信度阈值。

## 示性函数思维

示性函数就是把条件变成 `0/1` 或 `-1/1`。

它非常适合后面的路线。

```python
def indicator(x, threshold):
    return int(x > threshold)
```

可迁移到：

- 是否缺失。
- 是否异常。
- 是否触发交易信号。
- 是否需要人工审核。
- 是否满足输出格式。

Day10 的正负号例子也是这个思路：

```python
def sign_odd_even(x):
    if x % 2 == 1:
        return 1
    return -1
```

进一步可以优化成：

```python
sign = 2 * (x % 2 == 1) - 1
```

这就是：

```text
if/else 版本 -> 函数版本 -> 公式版本 -> 可迁移版本
```

## 算法优化顺序

不要一开始追高级写法。

按这个顺序：

```text
先写能看懂的版本。
再写少重复的版本。
再写函数封装版本。
最后才考虑性能。
```

推荐路线：

```text
if/else 版本
-> 函数版本
-> 字典映射版本
-> 数学公式版本
-> pandas / NumPy 向量化版本
```
