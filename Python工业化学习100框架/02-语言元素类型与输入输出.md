## 知识点 1：变量与类型

### 定义

变量是指向某个值的名字。类型决定这个值能做哪些操作。

### 为什么存在

类型能防止不合理操作，也能帮助程序判断数据能不能计算、比较、格式化或保存。

### 最小案例

```python
price: float = 102.5
shares: int = 3
symbol: str = "AAPL"
is_profitable: bool = True

market_value = price * shares
print(f"{symbol} 市值: {market_value:.2f}")
```

### 常见错误

- `"100" + 20` 会触发 `TypeError`。
- `input()` 返回的一定是 `str`。
- 浮点数不是精确小数。

### 工程应用

- CSV 和 API 数据经常需要从字符串转成数字。
- Agent 工具参数需要明确类型。
- Quant 计算必须确认价格、数量、权重都是数值。

### 未来扩展

- `decimal.Decimal`。
- `pydantic` 数据验证。
- 静态类型检查。

## 知识点 2：输入、转换、输出

### 定义

输入输出是程序和外部世界交互的方式。初学阶段常用 `input()` 和 `print()`。

### 为什么存在

程序只有能接收变化的输入，并输出可理解结果，才真正有用。

### 最小案例

```python
raw_price = input("价格: ")
raw_shares = input("数量: ")

price = float(raw_price)
shares = int(raw_shares)

print(f"持仓价值: {price * shares:.2f}")
```

### 常见错误

- 输入为空。
- 输入不是数字。
- 用 `int("10.5")` 直接转换小数字符串。

### 工程应用

- 命令行脚本。
- 数据导入工具。
- 配置读取。
- Agent 工具接口。

### 未来扩展

- `argparse`。
- `typer`。
- `logging`。

### 5 道递进题

#### 1. Easy - 变量操作计数

关联题：[LeetCode 2011](https://leetcode.com/problems/final-value-of-variable-after-performing-operations/)

题目：`++X` 加一，`--X` 减一，返回最终值。

讲解：字符串判断要服务业务规则，不要只背语法。

```python
# 输入: 操作字符串列表
# 输出: 最终整数值
def final_value(operations: list[str]) -> int:
    x = 0
    for op in operations:
        if "+" in op:
            x += 1
        else:
            x -= 1
    return x
```

#### 2. Easy - 数组合并

关联题：[LeetCode 1929](https://leetcode.com/problems/concatenation-of-array/)

题目：把数组复制一遍接到自己后面。

讲解：Python list 支持直接相加，先用清楚写法。

```python
# 输入: list[int]
# 输出: 原数组重复两次
def get_concatenation(nums: list[int]) -> list[int]:
    return nums + nums
```

#### 3. Medium - Richest Customer Wealth

关联题：[LeetCode 1672](https://leetcode.com/problems/richest-customer-wealth/)

题目：二维列表中，每行求和，返回最大和。

讲解：这就是最小版“客户资产汇总”。

```python
# 输入: accounts[i] 是第 i 个客户在多个账户的金额
# 输出: 最大客户总资产
def maximum_wealth(accounts: list[list[int]]) -> int:
    best = 0
    for row in accounts:
        best = max(best, sum(row))
    return best
```

#### 4. Medium - 价格字符串解析

题目：把 `"$123.45"`、`" 88.0 "` 转成 float。

讲解：数据分析第一步通常不是建模，而是把字段变干净。

```python
# 输入: 原始价格字符串
# 输出: float
# 边界: 空字符串返回 None，避免程序直接崩
def parse_price(text: str) -> float | None:
    cleaned = text.strip().replace("$", "").replace(",", "")
    if cleaned == "":
        return None
    return float(cleaned)
```

#### 5. Hard - 行情记录标准化

题目：把原始记录转成统一字段：`symbol`、`price`、`volume`。

讲解：这是以后 Quant 数据清洗和 Agent tool 输入校验的共同基础。

```python
# 输入: 可能字段不统一的 dict
# 输出: 标准 dict，方便后续计算
def normalize_quote(raw: dict) -> dict:
    return {
        "symbol": str(raw.get("symbol", "")).upper(),
        "price": parse_price(str(raw.get("price", ""))),
        "volume": int(raw.get("volume", 0)),
    }
```
