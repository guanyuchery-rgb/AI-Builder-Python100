### 基础优化补充：问题、场景与代码

**解决什么问题（Problem）**：正则解决的是“非结构化文本如何变成可计算字段”的问题。文件名、日志、报告片段和模型输出通常先是一段字符串，正则负责从里面提取日期、代码、金额、百分比或错误类型。

**真实科研场景**：从论文文件名、实验日志或模型输出文本里提取实验编号、指标名、结果值和日期，整理成后续能统计的表。

**Quant 场景**：从交易日志、券商导出文本或文件名中提取 ticker、日期、价格、收益率和订单编号。

**LLM/Agent 场景**：从 Agent 日志中提取 tool name、参数、错误信息和耗时，也可以校验 LLM 输出是否包含必要字段。

```python
import re


def parse_return_log(line):
    pattern = r"date=(\d{4}-\d{2}-\d{2}) ticker=([A-Z]+) return=([-+]?\d+\.\d+)%"
    match = re.search(pattern, line)
    if match is None:
        return None

    date, ticker, return_pct = match.groups()
    return {
        "date": date,
        "ticker": ticker,
        "return_pct": float(return_pct),
    }
```

## 基础详细讲解

### 如果你完全看不懂，先只抓这一句话

今天学的是：**正则就是从文本里按规则找东西。**

```text
一大段文本 -> 正则规则 -> 找出数字/日期/标签
```

不要一开始写复杂正则。先从“找数字”“找日期”开始。

### 正则不是普通搜索

普通搜索：找固定文本 `python`。

正则搜索：找“一类东西”，比如所有数字：

```python
import re
text = "price=102.5, volume=3000"
numbers = re.findall(r"\d+", text)
print(numbers)
```

这里 `\d+` 表示连续数字，所以会找到数字片段。

### 数据流追踪

| 代码 | 发生了什么 |
| --- | --- |
| `text = ...` | 准备原始文本 |
| `r"\d+"` | 定义匹配规则：连续数字 |
| `re.findall(...)` | 返回所有匹配结果组成的列表 |
| `print(numbers)` | 观察匹配是否合理 |

### 最容易错的点

| 现象 | 原因 | 怎么改 |
| --- | --- | --- |
| 匹配不到 | 规则太严格或写错 | 先写更简单的规则 |
| 匹配太多 | 规则太宽 | 增加边界或更具体的模式 |
| 小数被拆开 | 只写了整数规则 | 改成支持小数的规则 |
| 看不懂反斜杠 | 正则有特殊符号 | 先用 `r"..."` 原始字符串 |

### 正则表达式与文本解析

正则是从文本里提取模式的工具。初学只需要掌握少数稳定写法：数字、日期、邮箱样式、关键字段。不要一开始写很长的正则，先用简单模式拿到可验证结果。

### 基础知识深讲：模式匹配与文本边界

正则表达式解决的是模式匹配问题：从一段自由文本中找出符合规则的片段。它不是普通搜索，因为它搜索的不是固定字符串，而是一类模式，例如数字、日期、标签、代码块。

学习正则要先理解“匹配范围”。模式太宽，会抓到不该抓的内容；模式太窄，会漏掉合法内容。比如提取数字时，是否包含小数、负数、百分号，都要根据数据含义决定。正则没有脱离业务的正确答案，只有适合当前文本的规则。

分组是正则的重要能力。你可以匹配整段文本，但只提取其中一部分。例如从 `price=102.5` 中提取 `102.5`。这和后面解析日志、提取公告字段、检查模型输出格式非常相关。

正则代码必须配样例。至少准备正常样例、无匹配样例、容易误匹配样例。没有样例的正则很难维护，六个月后你很可能不知道当初为什么这么写。

### 手写顺序

- 先准备 3 行样例文本，明确要提取什么。
- 用 `re.findall()` 观察所有匹配，再决定是否需要分组。
- 正则结果一定要做样例检查，防止匹配过多或漏匹配。

### 今天必须真正弄懂

- **输入**：今天这段代码从哪里拿到数据，是手写值、终端输入、文件，还是函数参数。
- **处理**：每一行代码改变了什么状态，例如变量变了、列表多了一个元素、判断走了某个分支。
- **输出**：结果给谁看，是终端输出、返回值、文件，还是后面函数继续使用。

### 知识广度补充：正则要会读模式，不要只复制

正则最重要的基础符号：

- `\d`：数字。
- `+`：一个或多个。
- `*`：零个或多个。
- `.`：任意字符。
- `[]`：字符范围。
- `()`：捕获分组。
- `^` / `$`：开头和结尾。

```python
import re

text = "AAPL 2026-06-05 price=100.5"
match = re.search(r"([A-Z]+).*price=(\d+\.\d+)", text)
if match:
    ticker, price = match.groups()
    print(ticker, float(price))
```

工程习惯：先写最小样例，再逐步扩大匹配范围。不要一开始写一个特别长的正则。

### 带注释参考代码：Contains Duplicate

```python
def contains_duplicate(nums):
    seen = set()
    for num in nums:
        if num in seen:
            return True
        seen.add(num)
    return False
```
