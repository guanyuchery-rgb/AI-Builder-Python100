## 知识点 1：函数

### 定义

函数是有名字的代码块，接收输入，返回输出。

### 为什么存在

函数减少重复、表达意图、方便测试和复用。

### 最小案例

```python
def position_value(price: float, shares: int) -> float:
    return price * shares

value = position_value(102.5, 3)
print(f"持仓价值: {value:.2f}")
```

### 常见错误

- 忘记 `return`。
- 意外修改传入数据。
- 一个函数做太多事。

### 工程应用

- 指标计算。
- 数据清洗。
- API handler。
- Agent tool。

### 未来扩展

- docstring。
- 单元测试。
- 类型检查。
- 包结构。

## 知识点 2：模块与入口

### 定义

模块就是可以被导入的 Python 文件。`__main__` 保护语句用于区分“被导入”和“直接运行”。

### 为什么存在

同一个文件可以既被测试导入，又能作为脚本运行，不会产生不必要副作用。

### 最小案例

```python
def main() -> None:
    print("作为脚本运行")

if __name__ == "__main__":
    main()
```

### 常见错误

- 导入模块时自动执行代码。
- 文件名和标准库重名，例如 `random.py`。
- 循环导入。

### 工程应用

- CLI 工具。
- 项目工具函数。
- 可测试脚本。
- Agent 工具模块。

### 未来扩展

- `argparse`。
- `typer`。
- 项目打包。

## 现代代码示例：简单 CLI

```python
import argparse


def simple_return(start_price: float, end_price: float) -> float:
    return (end_price - start_price) / start_price


def main() -> None:
    parser = argparse.ArgumentParser(description="计算简单收益率")
    parser.add_argument("--start", type=float, required=True)
    parser.add_argument("--end", type=float, required=True)
    args = parser.parse_args()

    rate = simple_return(args.start, args.end)
    print(f"收益率: {rate:.2%}")


if __name__ == "__main__":
    main()
```

运行：

```bash
python3 return_cli.py --start 100 --end 108
```

### 5 道递进题

#### 1. Easy - 纯函数 add

题目：写函数 `add(a, b)`，只返回结果，不打印。

讲解：核心逻辑保持纯净，CLI 或 notebook 再负责展示。

```python
# 纯函数: 相同输入永远得到相同输出
def add(a: float, b: float) -> float:
    return a + b
```

#### 2. Easy - Valid Anagram

关联题：[LeetCode 242](https://leetcode.com/problems/valid-anagram/)

题目：判断两个字符串是否由相同字符组成。

讲解：函数边界小，适合写测试。

```python
from collections import Counter

# Counter 会统计每个字符出现次数
def is_anagram(s: str, t: str) -> bool:
    return Counter(s) == Counter(t)
```

#### 3. Medium - Roman to Integer

关联题：[LeetCode 13](https://leetcode.com/problems/roman-to-integer/)

题目：把罗马数字转成整数。

讲解：遇到小数在大数左边时做减法。

```python
# 如果当前值小于右边值，说明是 IV / IX 这类结构
def roman_to_int(s: str) -> int:
    values = {"I":1,"V":5,"X":10,"L":50,"C":100,"D":500,"M":1000}
    total = 0
    for i, ch in enumerate(s):
        value = values[ch]
        if i + 1 < len(s) and value < values[s[i + 1]]:
            total -= value
        else:
            total += value
    return total
```

#### 4. Medium - argparse 参数解析

题目：从命令行读取 `--symbol AAPL`。

讲解：以后你的 Quant/Agent 小工具都可以从这里长出来。

```python
import argparse

# parser 只处理输入边界，不放业务计算
def parse_args() -> argparse.Namespace:
    parser = argparse.ArgumentParser()
    parser.add_argument("--symbol", required=True)
    return parser.parse_args()
```

#### 5. Hard - Agent Tool 风格函数

题目：函数返回统一结构：成功时有 `data`，失败时有 `error`。

讲解：Agent 调工具时，最怕异常直接炸掉；统一返回更稳定。

```python
# 输入: symbol
# 输出: dict，适合作为 tool result
def get_quote_tool(symbol: str) -> dict:
    if not symbol:
        return {"ok": False, "error": "symbol is required"}
    # 这里先用 mock 数据，未来可替换成真实 API
    data = {"symbol": symbol.upper(), "price": 100.0}
    return {"ok": True, "data": data}
```
