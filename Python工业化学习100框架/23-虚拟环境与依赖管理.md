## 核心知识点：Environment Reproducibility

### 定义

Environment Reproducibility 是把今天能力固定成可复用模块的方式。

### 为什么存在

它让代码从“这次能跑”变成“下次能复查、能迁移、能被项目调用”。

### 最小案例

```python
# 终端执行
# python3 -m venv .venv
# source .venv/bin/activate
# python -m pip install pandas pytest
# python -m pip freeze > requirements.txt
```

### 常见错误

- 直接把逻辑写在 notebook 或临时脚本里。
- 输入字段没有校验。
- 结果只 print，不保存。
- 失败原因没有记录。

### 5 道递进题

#### 1. Easy - 标准化环境

题目：把原始 dict 转成统一字段。

讲解：统一字段名，后续函数才稳定。

```python
def normalize_env(raw: dict) -> dict:
    return {
        "name": str(raw.get("name", "")).strip(),
        "value": float(raw.get("value", 0) or 0),
        "tag": str(raw.get("tag", "default")).strip(),
    }
```

#### 2. Easy - 校验环境

题目：返回错误列表，而不是直接崩溃。

讲解：错误要可复盘。

```python
def validate_env(row: dict) -> list[str]:
    errors = []
    if not row.get("name"):
        errors.append("name is required")
    if row.get("value", 0) < 0:
        errors.append("value must be non-negative")
    return errors
```

#### 3. Medium - 批量汇总

题目：按 tag 汇总 value。

讲解：这是数据项目和日志分析的通用动作。

```python
from collections import defaultdict

def summarize_env(rows: list[dict]) -> dict[str, float]:
    summary = defaultdict(float)
    for row in rows:
        summary[row["tag"]] += row["value"]
    return dict(summary)
```

#### 4. Medium - 保存结果

题目：把 summary 保存成 JSON。

讲解：结果落盘，第二天才能继续。

```python
import json
from pathlib import Path

def save_env_summary(path: str, summary: dict) -> None:
    Path(path).write_text(json.dumps(summary, ensure_ascii=False, indent=2), encoding="utf-8")
```

#### 5. Hard - Tool 化

题目：输入原始 rows，返回 ok/data/errors。

讲解：这就是 Agent-ready 的最小工程形状。

```python
def env_tool(rows: list[dict]) -> dict:
    clean_rows = [normalize_env(row) for row in rows]
    errors = [validate_env(row) for row in clean_rows]
    flat_errors = [e for group in errors for e in group]
    if flat_errors:
        return {"ok": False, "errors": flat_errors}
    return {"ok": True, "data": summarize_env(clean_rows)}
```
