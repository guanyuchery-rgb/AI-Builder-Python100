## 基础详细讲解

### 如果你完全看不懂，先只抓这一句话

今天学的是：**列表推导式是 for 循环的短写，不是新魔法。**

普通写法：

```python
result = []
for x in values:
    result.append(x * 2)
```

推导式：

```python
result = [x * 2 for x in values]
```

它们做的是同一件事。

### 推导式怎么读

```python
[x * 2 for x in values]
```

按这个顺序读：

```text
for x in values  ->  每次拿一个 x
x * 2            ->  对 x 做处理
[...]            ->  把处理结果收集成列表
```

### 带 if 的推导式

```python
positive = [x for x in values if x > 0]
```

读法：

```text
遍历 values -> 只保留 x > 0 的元素 -> 放进新列表
```

### 什么时候不要用推导式

如果你写出来自己都读不懂，就不要用。

| 情况 | 建议 |
| --- | --- |
| 一层简单转换 | 可以用推导式 |
| 多个 if、多层循环 | 先用普通 for |
| 需要 Debug 中间变量 | 用普通 for |
| 给初学者看 | 普通 for 更清楚 |

### 列表推导式、迭代器与生成器

列表推导式是把“遍历 + 处理 + 收集结果”写得更紧凑。迭代器和生成器强调按需产生数据。零基础阶段先理解等价关系：推导式本质上还是 for 循环。

### 基础知识深讲：推导式背后的数据流

列表推导式的本质是数据流转换：从一个可遍历对象中取出元素，对每个元素做处理，再收集成新列表。`[x * 2 for x in values]` 等价于创建空列表、循环、append。先理解普通循环，再使用推导式，才不会把简洁写法变成黑盒。

推导式适合表达“一进一出”的简单变换，比如清洗字符串、提取字段、过滤正数。它不适合承载复杂业务规则。如果推导式里出现多层嵌套、多个条件、复杂函数调用，就应该退回普通循环，让逻辑更可读。

迭代器和生成器强调按需产生数据。列表会一次性把结果放进内存，生成器可以一个一个产出。初学阶段不需要写复杂生成器，但要知道它们为什么存在：大文件、日志流、批量任务都可能不能一次性全部加载。

推导式是写 Python 的常用能力，但不是炫技工具。工程代码的第一标准是可解释。你应该能把任何一个推导式翻译回普通 for 循环，并说清每一步输入和输出。

### 手写顺序

- 先写普通 for 循环版本，再改成列表推导式。
- 推导式只放一层简单逻辑；复杂判断仍然用普通循环。
- 生成器适合大文件或流式数据，先知道用途，不急着写复杂代码。

### 今天必须真正弄懂

- **输入**：今天这段代码从哪里拿到数据，是手写值、终端输入、文件，还是函数参数。
- **处理**：每一行代码改变了什么状态，例如变量变了、列表多了一个元素、判断走了某个分支。
- **输出**：结果给谁看，是终端输出、返回值、文件，还是后面函数继续使用。

### 和后面项目的关系

数据清洗、批量转换、特征生成会大量使用推导式；生成器会出现在日志和大文件处理中。

### 知识广度补充：推导式不要牺牲可读性

列表推导式适合短小转换，但不是越短越好。你还要知道：

- 列表推导式：生成列表。
- 字典推导式：生成字典。
- 集合推导式：生成集合。
- 生成器表达式：按需生成，不一次性保存所有结果。

```python
numbers = [1, 2, 3, 4]
squares = [n * n for n in numbers]
even_map = {n: n * n for n in numbers if n % 2 == 0}
unique = {n % 2 for n in numbers}
stream = (n * n for n in numbers)

print(squares)
print(even_map)
print(unique)
print(next(stream))
```

如果推导式里有太多判断和转换，退回普通 `for` 循环，代码会更适合长期维护。

- 解法 3：collections.Counter。

推荐先写最容易懂的版本。

再写复杂度更好的版本。

最后写一句：这个解法能迁移到今天哪个操作题。

### 带注释参考代码：Valid Anagram

```python
def is_anagram(s, t):
    if len(s) != len(t):
        return False
    counts = {}
    for ch in s:
        counts[ch] = counts.get(ch, 0) + 1
    for ch in t:
        if ch not in counts:
            return False
        counts[ch] -= 1
        if counts[ch] < 0:
            return False
    return True
```
