# 21 - 6月项目衔接路线

学习定位：完成 Day01-20 后，不继续无限学基础，而是进入项目。项目要小、稳、能跑、能复盘，服务你的长期方向：LLM、Agent、量化学习。

## 进入项目的最低标准

完成 Day01-20 后，如果你能做到下面这些，就可以开始项目：

- 能从终端运行 Python 脚本。
- 能读写 CSV、JSON、txt。
- 能写函数，并解释输入输出。
- 能处理基础异常。
- 能使用 dataclass 表示记录或配置。
- 能写 2-3 个简单测试。
- 能用 logging 记录过程。
- 能把结果保存为 JSON 或 Markdown。
- 能构造一个给 LLM 使用的 prompt。
- 能说明 Agent tool 的输入、输出和失败情况。

## 项目 1：Quant Research Starter

目标：做一个最小量化研究脚本。

输入：

- `returns.csv`
- 字段：`symbol,date,return_rate`

输出：

- 每个 symbol 的平均收益。
- 最好和最差 symbol。
- 简化波动率。
- JSON summary。
- Markdown 报告。

核心能力：

- 文件读写。
- CSV 清洗。
- 指标函数。
- 测试。
- 日志。

完成标准：

- 可以一条命令运行。
- 可以换 CSV 继续跑。
- 输出结果能被 LLM 解释。

## 项目 2：LLM Report Assistant

目标：做一个本地报告辅助工具。

输入：

- 一段 Markdown 或 txt。
- 一份 JSON summary。

输出：

- 给 LLM 的结构化 prompt。
- 摘要模板。
- 风险提示清单。

核心能力：

- 文本处理。
- JSON 结构化。
- Prompt 设计。
- 输出保存。

完成标准：

- 不直接依赖复杂框架。
- 能把 Quant summary 转成清晰 prompt。
- 保留输入来源。

## 项目 3：Agent Tool Prototype

目标：把一个 Python 函数包装成 Agent-ready tool。

输入：

- `input_path`
- `metric`
- `output_path`

输出：

- `summary`
- `errors`
- `next_action`

核心能力：

- 函数边界。
- 类型标注。
- dataclass。
- 参数验证。
- 错误返回。
- JSON 输出。

完成标准：

- 工具不依赖 UI。
- 输入错误时不崩溃。
- 输出可被 LLM 或 Agent 继续使用。

## 不继续学基础的信号

如果你已经能把 CSV 读进来、清洗、计算、保存、生成 prompt，就不要继续无限学语法。下一步应该做项目，因为项目会自然暴露你缺什么。

## 复盘问题

1. 我最稳定的 3 个 Python 能力是什么？
2. 我最容易出错的是路径、类型、函数、还是数据结构？
3. 我能否独立做一个 CSV -> JSON -> prompt 的流程？
4. 我的第一个 Quant 项目输入和输出是什么？
5. 哪个函数最适合先包装成 Agent tool？
