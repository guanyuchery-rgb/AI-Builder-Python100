## 基础详细讲解

### 如果你完全看不懂，先只抓这一句话

今天学的是：**测试就是把“我期望函数返回什么”写成代码。**

```text
给函数一个输入 -> 得到实际输出 -> 和期望输出比较
```

### assert 怎么读

```python
def add(a, b):
    return a + b

assert add(2, 3) == 5
```

这句话意思是：我期望 `add(2, 3)` 的结果等于 5。如果不是 5，测试失败。

### 为什么先测函数

函数最适合测试，因为它通常是：

```text
输入 -> 处理 -> 返回值
```

比如：

```python
def average(values):
    if not values:
        return 0
    return sum(values) / len(values)
```

可以测试：

```python
assert average([1, 2, 3]) == 2
assert average([]) == 0
```

### 测试失败怎么看

测试失败不是坏事。它告诉你：实际结果和期望结果不一样。

| 你要看 | 说明 |
| --- | --- |
| 输入是什么 | 哪个样例触发失败 |
| 期望是什么 | assert 右边写的值 |
| 实际是什么 | 函数真实返回值 |
| 谁错了 | 可能是函数错，也可能是期望写错 |

### 最小测试习惯

每个重要函数至少写三类：

- 正常输入。
- 空值或边界输入。
- 容易写错的输入。

### pytest 与最小测试

测试是把你的预期写成代码。它不是大公司才需要的东西，而是让你确认函数在修改后仍然可靠。最小测试只需要给定输入、调用函数、断言输出。

### 基础知识深讲：测试是可执行预期

测试的核心不是 pytest 语法，而是“把预期写成代码”。当你写 `assert average([1, 2, 3]) == 2`，你是在告诉项目：这个函数在这个输入下必须返回 2。以后代码改动后，测试能自动检查这个预期是否仍然成立。

最适合先测试的是纯函数。纯函数给定输入就返回输出，不依赖终端输入、文件、网络或时间。比如平均值、最大值、收益率计算都适合测试。越纯的函数，越容易测试，也越容易复用。

测试用例要覆盖正常、边界和错误。正常输入证明功能能跑；边界输入证明你考虑了空值、零、单元素；错误输入证明失败方式可控。测试不是为了追求数量，而是为了覆盖关键风险。

失败的测试是一种反馈，不是麻烦。它告诉你实际输出和预期输出不一致。调试时不要急着改代码，先确认是测试预期错了，还是函数逻辑错了。这个判断能力会影响后面所有工程项目。

### 手写顺序

- 先测试纯函数，例如 `add(2, 3) == 5`。
- 每个测试只验证一个行为，不要把很多逻辑塞进一个测试。
- 至少写正常输入、边界输入、错误输入三类案例。

### 今天必须真正弄懂

- **输入**：今天这段代码从哪里拿到数据，是手写值、终端输入、文件，还是函数参数。
- **处理**：每一行代码改变了什么状态，例如变量变了、列表多了一个元素、判断走了某个分支。
- **输出**：结果给谁看，是终端输出、返回值、文件，还是后面函数继续使用。

### 和后面项目的关系

Quant 指标、清洗函数、Agent tool 都需要测试，否则项目越改越不敢动。

#### 这一天到底在练什么

测试不是额外负担，而是把“我认为这个函数应该怎样”写下来。函数改动后，测试能帮你确认旧能力还在。

#### 手写时按这个顺序

#### 最常见卡点

测试失败不等于坏事，它是在告诉你预期和实际不一致。先看失败的输入、实际输出和期望输出。

### 知识广度补充：测试要覆盖正常和异常

最小测试不只测“正确输入”，还要测边界：

- 正常输入。
- 空输入。
- 类型不对。
- 边界值，例如 0、负数、最大值。
- 预期会报错的情况。

```python
def divide(a, b):
    if b == 0:
        raise ValueError("b cannot be zero")
    return a / b


def test_divide_normal():
    assert divide(4, 2) == 2


def test_divide_zero():
    import pytest
    with pytest.raises(ValueError):
        divide(4, 0)
```

测试不是为了显得专业，而是给未来修改留一条安全线。

### 带注释参考代码：Top K Frequent Elements

```python
def top_k_frequent(nums, k):
    counts = {}
    for num in nums:
        counts[num] = counts.get(num, 0) + 1
    items = sorted(counts.items(), key=lambda item: item[1], reverse=True)
    return [num for num, count in items[:k]]
```
