## 基础详细讲解

### 脚本、终端与 print 输出

第一天只建立一个模型：Python 文件是输入给解释器的指令清单，终端是你观察结果和错误的位置。`print()` 不是为了好看，而是最小的观察工具；你先让程序把中间状态说出来，后面才能调试数据、接口、模型和 Agent tool。

### 手写顺序

- 先在一个固定文件夹里新建 `.py` 文件，再运行 `python3 文件名.py`。不要一开始就在交互窗口里反复粘贴，因为文件更容易复查。
- 写一行就运行一次，确认输出符合预期；如果报错，先看报错最后一行和行号。
- 把文字放进引号，把计算留在引号外。`print("2+3")` 输出文字，`print(2+3)` 输出计算结果。

### 今天必须真正弄懂

- **输入**：今天这段代码从哪里拿到数据，是手写值、终端输入、文件，还是函数参数。
- **处理**：每一行代码改变了什么状态，例如变量变了、列表多了一个元素、判断走了某个分支。
- **输出**：结果给谁看，是终端输出、返回值、文件，还是后面函数继续使用。

### 和后面项目的关系

以后任何项目都有入口文件，例如 `main.py`、`app.py` 或 `cli.py`。今天的能力就是确认入口文件能跑、输出能看懂、错误能定位。

## 5 道基础巩固题

#### 1. 打印三行文字

题目：连续打印学习方向、今天目标、完成标准。

讲解：只练习 print，不使用变量。

```python
print("学习方向：AI Builder Python")
print("今天目标：运行第一个脚本")
print("完成标准：终端看到输出")
```

#### 2. 打印简单计算

题目：打印加法、减法、乘法结果。

讲解：确认 Python 能直接做基础计算。

```python
print(10 + 5)
print(10 - 5)
print(10 * 5)
```

#### 3. 区分文字和计算

题目：分别打印 `"2 + 3"` 和 `2 + 3`。

讲解：理解引号里的内容是字符串。

```python
print("2 + 3")
print(2 + 3)
```

#### 4. 输出带说明的结果

题目：先打印说明，再打印数字。

讲解：公开课程里的输出要让读者看得懂。

```python
print("今日学习分钟数：")
print(120)
```

#### 5. 最小 Debug 记录

题目：写一个错误示例和修复示例。

讲解：第一天只要学会记录错误，不要求完全理解。

```python
# 错误示例：print("hello)
# 修复示例：
print("hello")
```
