## 知识点 1：Regex 的定位

### 定义

正则表达式 Regex 是一种文本模式匹配语言，用来查找、提取、验证或替换文本。

### 为什么存在

真实数据不总是整齐表格。日志、公告、邮件、网页文本、LLM 输出都需要解析。

### 最小案例

```python
import re

text = "AAPL price=102.5 volume=3000"
match = re.search(r"price=(\\d+\\.\\d+)", text)

if match:
    print(float(match.group(1)))
```

### 常见错误

- 忘记使用 raw string：`r"..."`
- 正则写得太贪婪。
- 用 Regex 解析本该用 JSON/HTML parser 解析的数据。

### 工程应用

- 日志解析。
- 表单验证。
- LLM 输出清洗。
- 金融公告字段提取。

### 未来扩展

- 命名分组。
- 编译正则。
- 文本解析库。
- NLP。

## 知识点 2：提取结构化字段

### 定义

字段提取是把文本中的关键信息转成 dict、list 或表格。

### 为什么存在

LLM、Agent 和数据分析都更擅长处理结构化数据。

### 最小案例

```python
import re

line = "trade: symbol=AAPL, price=102.5, quantity=10"
pattern = r"symbol=(?P<symbol>[A-Z]+), price=(?P<price>\\d+\\.\\d+), quantity=(?P<quantity>\\d+)"

match = re.search(pattern, line)
if match:
    record = match.groupdict()
    record["price"] = float(record["price"])
    record["quantity"] = int(record["quantity"])
    print(record)
```

### 常见错误

- 没检查 `match is None` 就调用 `group()`。
- 提取后忘记类型转换。
- 模式只适配一个样本，换一行就坏。

### 工程应用

- 交易日志转 CSV。
- Prompt 输出格式校验。
- 错误日志分类。
- 文本数据预处理。

### 未来扩展

- parser function。
- 数据验证。
- LLM structured output。

## Debug 日志

- `AttributeError: 'NoneType' object has no attribute 'group'`：没有匹配成功。
- 匹配过多：检查 `.*` 是否太贪婪。
- 匹配不到中文：检查字符范围和编码。


## 面试角度 Interview

Regex 适合轻量文本模式提取，但不适合解析复杂嵌套结构。复杂 JSON、HTML、PDF 应该使用专门解析器。

## Quant 关联

新闻、公告、研报标题、交易日志都可能先以文本形式出现。Regex 能把文本变成可分析字段。

## LLM / Agent 关联

LLM 输出有时不稳定。Regex 可以作为轻量校验和兜底清洗工具，但更稳定的方向是 structured output。

### 5 道递进题

#### 1. Easy - 提取邮箱

题目：从文本中提取邮箱地址。

讲解：正则适合模式明确的文本抽取。

```python
import re

def extract_emails(text: str) -> list[str]:
    # \w 表示字母数字下划线；点号在字符类里不需要转义
    pattern = r"[\w.-]+@[\w.-]+\.\w+"
    return re.findall(pattern, text)
```

#### 2. Easy - 提取股票代码

题目：提取 `$AAPL`、`$MSFT` 这种 ticker。

讲解：Quant 文本数据经常需要识别标的。

```python
import re

def extract_tickers(text: str) -> list[str]:
    # \b 表示单词边界，避免把更长字符串误匹配
    return re.findall(r"\$[A-Z]{1,5}\b", text)
```

#### 3. Medium - 日期格式校验

题目：判断字符串是否是 `YYYY-MM-DD`。

讲解：正则只校验形状，真实日期还要用 datetime。

```python
import re

def looks_like_date(text: str) -> bool:
    # fullmatch 要求整个字符串都匹配
    return re.fullmatch(r"\d{4}-\d{2}-\d{2}", text) is not None
```

#### 4. Medium - 解析日志行

题目：解析 `2026-06-01 ERROR timeout`。

讲解：命名分组让输出天然接近 dict。

```python
import re

def parse_log_line(line: str) -> dict | None:
    pattern = r"(?P<date>\d{4}-\d{2}-\d{2})\s+(?P<level>INFO|ERROR|WARN)\s+(?P<message>.*)"
    match = re.fullmatch(pattern, line)
    if not match:
        return None
    return match.groupdict()
```

#### 5. Hard - Prompt 脱敏

题目：隐藏邮箱和手机号后再发给 LLM。

讲解：LLM/Agent 项目要养成隐私边界意识。

```python
import re

def redact_pii(text: str) -> str:
    # 先替换邮箱，再替换手机号
    text = re.sub(r"[\w.-]+@[\w.-]+\.\w+", "<EMAIL>", text)
    text = re.sub(r"\b\d{3}[- ]?\d{3,4}[- ]?\d{4}\b", "<PHONE>", text)
    return text
```
