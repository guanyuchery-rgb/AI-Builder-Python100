## 知识点 1：条件判断

### 定义

分支结构让程序根据条件选择不同路径。

### 为什么存在

真实程序必须处理不同输入、异常状态、权限、阈值和业务规则。

### 最小案例

```python
cash = 1000
order_value = 1200

if order_value <= cash:
    print("订单通过")
else:
    print("现金不足")
```

### 常见错误

- 把 `==` 写成 `=`。
- 用字符串和数字比较。
- 忘记等于边界。

### 工程应用

- 输入验证。
- 文件是否存在。
- API 参数检查。
- 风险阈值控制。

### 未来扩展

- 早返回。
- 异常处理。
- `match` 模式匹配。
- 数据验证库。

## 知识点 2：早返回

### 定义

早返回是在函数中尽早处理无效情况，让主逻辑更清楚。

### 为什么存在

它能减少嵌套，让验证逻辑更容易读、测、改。

### 最小案例

```python
def can_place_order(cash: float, order_value: float) -> bool:
    if order_value <= 0:
        return False
    if cash < order_value:
        return False
    return True

print(can_place_order(1000, 800))
```

### 常见错误

- 返回 `"yes"` / `"no"` 而不是 `True` / `False`。
- 把验证和打印混在一起。
- 忘记负数、零、等于边界。

### 工程应用

- 数据清洗过滤。
- API 请求验证。
- Agent 工具安全检查。
- Quant 风控规则。

### 未来扩展

- 自定义异常。
- `pydantic` validator。
- 单元测试。

#### 1. Easy - Fizz Buzz

关联题：[LeetCode 412](https://leetcode.com/problems/fizz-buzz/)

题目：按规则输出 `Fizz`、`Buzz`、`FizzBuzz` 或数字。

讲解：分支顺序很重要，先判断最特殊情况。

```python
# 输入: n
# 输出: 长度为 n 的字符串列表
def fizz_buzz(n: int) -> list[str]:
    ans = []
    for i in range(1, n + 1):
        if i % 15 == 0:
            ans.append("FizzBuzz")
        elif i % 3 == 0:
            ans.append("Fizz")
        elif i % 5 == 0:
            ans.append("Buzz")
        else:
            ans.append(str(i))
    return ans
```

#### 2. Easy - 回文数字

关联题：[LeetCode 9](https://leetcode.com/problems/palindrome-number/)

题目：判断整数正读倒读是否相同。

讲解：先用字符串版本掌握边界，再学数学版。

```python
# 负数带负号，不算回文
def is_palindrome_number(x: int) -> bool:
    if x < 0:
        return False
    text = str(x)
    return text == text[::-1]
```

#### 3. Medium - 有效括号

关联题：[LeetCode 20](https://leetcode.com/problems/valid-parentheses/)

题目：判断括号是否正确闭合。

讲解：stack 是工业代码里很常见的“等待匹配”结构。

```python
# 输入: 只包含括号的字符串
# 输出: bool
def is_valid_parentheses(s: str) -> bool:
    pairs = {")": "(", "]": "[", "}": "{"}
    stack = []
    for ch in s:
        if ch in pairs.values():
            stack.append(ch)
        elif not stack or stack.pop() != pairs[ch]:
            return False
    return len(stack) == 0
```

#### 4. Medium - 订单输入验证

题目：订单数量必须大于 0，价格必须大于 0，方向只能是 `BUY` 或 `SELL`。

讲解：真实系统里，坏输入比复杂算法更常见。

```python
# 输出: (是否有效, 错误原因)
def validate_order(side: str, quantity: int, price: float) -> tuple[bool, str]:
    if side not in {"BUY", "SELL"}:
        return False, "side must be BUY or SELL"
    if quantity <= 0:
        return False, "quantity must be positive"
    if price <= 0:
        return False, "price must be positive"
    return True, "ok"
```

#### 5. Hard - 风险信号判断

题目：根据收益率、波动率、最大回撤输出 `LOW/MEDIUM/HIGH`。

讲解：把统计指标变成规则，是 Quant 项目的常见第一版。

```python
# 输入: 三个已经算好的指标
# 输出: 风险等级字符串
def risk_level(ret: float, volatility: float, max_drawdown: float) -> str:
    if max_drawdown <= -0.20 or volatility >= 0.35:
        return "HIGH"
    if max_drawdown <= -0.10 or volatility >= 0.20 or ret < 0:
        return "MEDIUM"
    return "LOW"
```
