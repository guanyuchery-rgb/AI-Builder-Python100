## What

LLM 结构化报告基础 是一个知识模块。

它帮助你把输入、处理、输出、错误边界和人工审查拆清楚。

## Why

如果没有这层认知，后面进入 IC 时容易把功能堆在一起：输入没定义、输出不稳定、错误无法复现、结果无法解释。

Day 主线先建立认知地图，IC 再做工程验证。

## How

最小学习流程：

1. 写清输入。
2. 写清输出。
3. 写清失败点。
4. 写一个最小函数。
5. 用虚拟样例跑通。

```python
# Day59 - LLM report boundary 最小示例

def check_boundary(data):
    if not data:
        return {"ok": False, "error": "empty data"}

    return {
        "ok": True,
        "input_type": type(data).__name__,
        "review": "check input, output, and failure point",
    }

print(check_boundary({"sample": "demo"}))
```

## Common Errors

| 错误 | 表现 | Debug 方法 |
| --- | --- | --- |
| 输入没有样例 | 不知道函数处理什么 | 先写 1 条虚拟样例 |
| 输出没有结构 | 后续步骤接不上 | 固定返回字段 |
| 一次写太大 | 当天超过 4 小时 | 放进 IC 验证 |
| 只读不跑 | 以为懂了但不会改 | 保留最小可运行代码 |
