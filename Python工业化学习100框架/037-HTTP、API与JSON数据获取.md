### 1. 是什么（What）

HTTP、API 与 JSON 数据获取 是用 Python 解决工程任务时的一组小工具。最简定义：它帮助你把「请求 API 并解析 JSON、urllib/request timeout JSON、输入输出边界」放进可运行流程里。先把它当成解决问题的方法，不要先背术语。

#### 3. 真实项目里怎么用

- **数据分析**：先明确字段、类型、缺失值、样本范围，再写处理逻辑。不要直接对未知数据写复杂代码。

- **LLM/Agent**：所有输入输出都要有结构，模型结果要校验，高风险动作要有人类审查点。
- **开源作品集**：代码之外还要有 README、样例输入、运行命令、测试和复盘说明。

#### 4. 常见坑和边界

- CSV 读出来通常都是字符串，要显式转换数字。
- JSON 结构不要嵌套过深，否则后续校验困难。
- 字段名要稳定，避免今天叫 price 明天叫 close。
- 不要把所有逻辑堆进一个文件或一个函数。先拆出最小核心函数，再决定是否需要 CLI、API、看板或测试。
- 不要用“能跑一次”代表掌握。真正掌握是换一份输入、换一个场景、换一个项目后仍然能迁移。

### 带注释参考代码：Two Sum

```python
def two_sum(nums, target):
    seen = {}  # 值 -> 下标
    for i, num in enumerate(nums):
        need = target - num
        if need in seen:
            return [seen[need], i]
        seen[num] = i
    return []
```

## 强化练习与工程使用

> 所属阶段：应用接口、自动化与交付。目标是：把脚本包装成可复用工具或轻量服务。

### 工程使用场景

今天的主题是 **HTTP、API 与 JSON 数据获取**，不要只把它当成一个孤立知识点。它应该落到 `DataTool 交付` 里的一个可复用能力：输入清楚、处理过程可检查、输出可以保存，失败时能解释原因。重点检查：输入参数、失败重试、运行命令、交付说明。

### 强化 7 题

1. **复现题**：不看答案重写今天最小案例，并写出运行命令和预期输出。
2. **边界题**：构造 3 个异常输入，说明程序应该返回结果、跳过、还是报错。
3. **数据题**：准备 5-10 条虚拟样例数据，覆盖正常值、空值、重复值和极端值。
4. **函数题**：把核心逻辑封装成 1 个函数，函数名、参数名和返回值必须能读懂。
5. **测试题**：写 2 个正常测试和 1 个边界测试，确认修改后行为不变。
6. **算法题**：完成 [Valid Anagram](https://leetcode.com/problems/valid-anagram/)，题型：字符串 / 计数。提示：用计数字典比较两个输入，适合练字段统计和文本清洗。
7. **项目题**：把今天结果接入 `DataTool 交付` 的一个小模块，并在 README 里写 3 行使用说明。

### 难度控制

Medium 起步，重点是把算法思维放进 CLI、API、重试和自动化任务。 每个练习都要留下运行命令和失败样例，像交付小工具一样处理。 如果算法题暂时写不出完整代码，先写暴力思路、输入输出样例和复杂度，再补优化版。
