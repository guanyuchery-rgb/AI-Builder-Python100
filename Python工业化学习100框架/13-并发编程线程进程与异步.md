## 知识点 1：并发与并行

### 定义

并发 Concurrency 是让多个任务在同一时间段推进；并行 Parallelism 是多个任务真正同时执行。

### 为什么存在

下载、API 请求、文件 I/O 经常在等待。并发可以减少等待浪费。

### 最小案例

```python
from concurrent.futures import ThreadPoolExecutor
from time import sleep


def fetch_symbol(symbol: str) -> str:
    sleep(1)
    return f"{symbol}: done"


symbols = ["AAPL", "MSFT", "NVDA"]

with ThreadPoolExecutor(max_workers=3) as executor:
    results = list(executor.map(fetch_symbol, symbols))

print(results)
```

### 常见错误

- 以为线程一定能加速 CPU 密集计算。
- 并发任务共享可变数据导致混乱。
- 没有限制并发数量。

### 工程应用

- 批量下载行情。
- 批量调用 API。
- 文件处理任务。
- Agent 多工具任务。

### 未来扩展

- multiprocessing。
- asyncio。
- 队列。
- 任务调度。

## 知识点 2：asyncio

### 定义

`asyncio` 是 Python 的异步 I/O 框架，适合大量等待型任务。

### 为什么存在

当任务主要在等待网络或 I/O，异步可以用较少资源推进多个任务。

### 最小案例

```python
import asyncio


async def fetch(symbol: str) -> str:
    await asyncio.sleep(1)
    return f"{symbol}: done"


async def main() -> None:
    symbols = ["AAPL", "MSFT", "NVDA"]
    results = await asyncio.gather(*(fetch(symbol) for symbol in symbols))
    print(results)


asyncio.run(main())
```

### 常见错误

- 忘记 `await`。
- 在已有 event loop 中乱用 `asyncio.run()`。
- 把 CPU 密集任务放进 asyncio 期待变快。

### 工程应用

- 高并发 API 请求。
- Agent 多工具调用。
- Web 服务异步 handler。

### 未来扩展

- `httpx` async client。
- async queue。
- FastAPI async endpoint。

## Debug 日志

- 并发后结果顺序不符合预期：明确是否需要保序。
- 速度没变快：判断任务是 I/O bound 还是 CPU bound。
- 任务失败没看到错误：检查 future/result 异常。

### 5 道递进题

#### 1. Easy - ThreadPool 并发 map

题目：并发处理多个 symbol。

讲解：I/O 等待型任务适合线程池。

```python
from concurrent.futures import ThreadPoolExecutor

def normalize_symbol(symbol: str) -> str:
    # 每个任务只做一件事，方便并发调用
    return symbol.strip().upper()

def normalize_many(symbols: list[str]) -> list[str]:
    with ThreadPoolExecutor(max_workers=4) as pool:
        return list(pool.map(normalize_symbol, symbols))
```

#### 2. Easy - retry 包装

题目：函数失败时最多重试 3 次。

讲解：网络/API 调用必须默认考虑临时失败。

```python
def retry_call(func, attempts: int = 3):
    last_error = None
    for _ in range(attempts):
        try:
            return func()
        except Exception as error:
            last_error = error
    raise RuntimeError("all retry attempts failed") from last_error
```

#### 3. Medium - async gather

题目：异步执行多个协程并收集结果。

讲解：asyncio 适合大量 API 请求，但要先理解事件循环。

```python
import asyncio

async def fake_fetch(symbol: str) -> dict:
    # sleep 模拟网络等待
    await asyncio.sleep(0.1)
    return {"symbol": symbol, "price": 100.0}

async def fetch_many(symbols: list[str]) -> list[dict]:
    tasks = [fake_fetch(symbol) for symbol in symbols]
    return await asyncio.gather(*tasks)
```

#### 4. Medium - CPU 任务进程池

题目：并行计算多个数字平方。

讲解：CPU 密集任务优先考虑进程池，不是线程池。

```python
from concurrent.futures import ProcessPoolExecutor

def square(n: int) -> int:
    return n * n

def square_many(nums: list[int]) -> list[int]:
    with ProcessPoolExecutor() as pool:
        return list(pool.map(square, nums))
```

#### 5. Hard - 限速请求队列

题目：每批最多处理 `batch_size` 个任务。

讲解：API rate limit 是 Quant 和 LLM 工程都会遇到的边界。

```python
def process_in_batches(items: list[str], batch_size: int) -> list[str]:
    result = []
    for i in range(0, len(items), batch_size):
        # 每次只处理一小批，避免打爆外部服务
        batch = items[i:i + batch_size]
        result.extend(normalize_many(batch))
    return result
```
