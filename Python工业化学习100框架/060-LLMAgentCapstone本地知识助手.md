### 带注释参考代码：Number of Islands

```python
def num_islands(grid):
    rows, cols = len(grid), len(grid[0])

    def dfs(r, c):
        if r < 0 or r >= rows or c < 0 or c >= cols:
            return
        if grid[r][c] != "1":
            return
        grid[r][c] = "0"
        dfs(r + 1, c)
        dfs(r - 1, c)
        dfs(r, c + 1)
        dfs(r, c - 1)

    count = 0
    for r in range(rows):
        for c in range(cols):
            if grid[r][c] == "1":
                count += 1
                dfs(r, c)
    return count
```

## 强化练习与工程使用

> 所属阶段：LLM / RAG / Agent 基础。目标是：把模型调用变成可评估、可回放的工程流程。

### 工程使用场景

今天的主题是 **LLM/Agent 本地知识助手原理**，不要只把它当成一个孤立知识点。它应该落到 `本地知识助手` 里的一个可复用能力：输入清楚、处理过程可检查、输出可以保存，失败时能解释原因。重点检查：上下文来源、结构化输出、错误恢复、人工审查点。

### 强化 7 题

1. **复现题**：不看答案重写今天最小案例，并写出运行命令和预期输出。
2. **边界题**：构造 3 个异常输入，说明程序应该返回结果、跳过、还是报错。
3. **数据题**：准备 5-10 条虚拟样例数据，覆盖正常值、空值、重复值和极端值。
4. **函数题**：把核心逻辑封装成 1 个函数，函数名、参数名和返回值必须能读懂。
5. **测试题**：写 2 个正常测试和 1 个边界测试，确认修改后行为不变。
6. **算法题**：完成 [Reverse Linked List](https://leetcode.com/problems/reverse-linked-list/)，题型：链表。提示：理解指针更新顺序，训练状态转移。

### 难度控制

Medium 为主，逐步接触图、堆、检索和结构化解析。 把题目理解成 LLM/Agent 的工具能力：输入契约、输出结构、可验证结果。 如果算法题暂时写不出完整代码，先写暴力思路、输入输出样例和复杂度，再补优化版。
