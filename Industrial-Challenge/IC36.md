## Part B - 工程挑战：策略实验配置器

目标：只使用 Day71-Day72 附近已经学过的知识，做一个能运行、能解释、能提交的小工具。

### 输入

- 一份虚拟样例数据，或者 3-5 条手写记录。
- 不使用真实隐私数据。

### 输出

- 一个可运行脚本或小项目。
- 一份运行结果说明。
- 一份错误记录或 Debug 记录。

### 最小功能

1. 明确输入字段和数据结构。
2. 写出核心函数或核心流程。
3. 处理至少 1 个错误或边界情况。
4. 输出人能看懂的结果。
5. 写 3-5 行 README 说明运行方式。

### 限制

- 不提前引入高级 API。
- 不为了工程化而复杂化。
- 当前 IC 只使用最近 Day 学过的知识。
